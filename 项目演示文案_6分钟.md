# “问阶”AI 学习工作台项目演示文案（约 6 分钟）

> 建议语速：每分钟 220～250 字。方括号内容为操作提示，不需要念出。

> 更重要的是，系统会根据持续对话和学习行为，动态更新学生的专业基础、学习目标、薄弱点和内容偏好，形成八维学习画像。这个画像将进一步驱动后续的路径规划和个性化推送，从而实现越使用越了解学生。

## 1:15—2:05 智能答疑与课程知识库

[操作：进入“对话”，输入预设问题，例如“请结合教材解释支持向量机中核函数的作用，并告诉我该先补哪些知识”。]

接下来演示智能答疑。这里输入一个课程问题。回答时，系统不仅读取当前对话，还会结合学习画像和用户上传的课程资料。

[操作：展示回答中的教材名称、PDF 页码或“AI 扩展”标记。]

可以看到，回答会标注具体课程文档和 PDF 页码；如果某部分超出本地教材覆盖范围，系统会明确标记为“AI 扩展”。课程 PDF 上传后，系统会自动识别文本层；对于扫描版资料，还可以通过 OCR 提取内容，并按页切分建立本地知识库。这样既能让回答贴合本校课程，又能降低模型幻觉。

此外，系统支持文字、图片和语音等多模态交互，回答可以流式呈现，并完整渲染公式、表格和代码，满足不同课程场景下的答疑需求。

## 2:05—3:15 多智能体资源生成

[操作：进入“资源”页面，选择一个主题；勾选知识文档、题库、实操案例、思维导图等 Agent，点击生成。建议提前准备好已生成结果。]

一次回答解决的是当前问题，但系统还需要把知识转化为可以继续学习的资源。我们以“机器学习模型评估”为例，选择生成知识文档、练习题、实操案例和思维导图。

系统采用真实的多智能体协作流程。首先，需求分析 Agent 会独立判断学习目标、当前短板和资源难度；随后，各类资源 Agent 并行工作，分别生成文档、题库、案例、导图等七类学习资源；最后，审核整合 Agent 检查内容质量、事实依据和资源之间的衔接关系。

[操作：展开协作过程或生成结果，指向“AI 生成资源”“检索资源·可溯源”等标识。]

每个 Agent 的执行状态、中间产物和审核反馈都可以追踪；单个 Agent 失败不会阻断其他任务。对于外部教育资源，系统优先从国家高等教育智慧教育平台、中国大学 MOOC、学堂在线、MIT OCW 等可信目录检索，并展示来源机构、链接和可信度。同时，界面会明确区分“可溯源检索资源”和“AI 生成资源”，避免混淆来源。

## 3:15—4:10 个性化学习路径与精准推送

[操作：进入“路径”页面，切换知识大类，展开阶段和 Todo。]

资源生成完成后，系统不会把它们堆成一个列表，而是根据学生画像、同类历史需求、错题和当前进度，组织成一条个性化学习路径。

路径按照知识大类独立维护。比如机器学习、数据结构和编译原理可以分别拥有自己的路径，互不覆盖。每条路径包含阶段目标、建议时长、学习顺序、可执行 Todo、掌握证据和推荐资源。学生完成任务后可以直接勾选，系统会计算进度并给出下一步提示。

[操作：进入“推送”页面，展示按阶段分组的资源及推荐理由。]

在精准推送页面，系统会把实际生成的文档、视频、题库、实操和导图关联到对应学习阶段，并说明“为什么现在推荐给你”。当同一主题出现新的提问、错题或学习进展时，系统会对原路径增量调整，而不是简单清空重建。

## 4:10—5:00 练习评估与闭环更新

[操作：打开评估或题库，展示答题结果、错题和复盘建议。]

学习是否有效，最终需要证据验证。学生完成练习后，系统会记录正确率、薄弱知识点和错题内容，并给出针对性的解析与复盘建议。错题可以沉淀到错题本，成为下一轮资源生成和路径调整的重要依据。

这就形成了问阶的核心闭环：学生提出需求，系统理解学习状态；多智能体生成并审核资源；路径把资源转化为行动；练习和使用行为形成学习证据；这些证据再反向更新画像、资源优先级和下一阶段路径。

因此，系统关注的不只是“有没有生成内容”，还关注学生是否真正使用、是否掌握，以及下一步应该如何调整。

## 5:00—5:35 内容沉淀、社区与多端体验

[操作：进入“存储”或“社区”页面；如条件允许，快速展示移动端界面。]

生成的知识文档可以下载为 Markdown，思维导图可以导出为 SVG，并按照知识大类自动归档到个人存储中。优质内容还可以发布到学习社区，经过交流、收藏和复用，从个人学习成果进一步沉淀为共享资源。

系统同时适配电脑和移动端，学生可以在电脑上完成深度学习和资源整理，也可以在手机上查看路径、复习内容和参与社区讨论，让学习过程保持连续。

## 5:35—6:00 总结

[操作：回到首页或项目总览页。]

最后总结一下。问阶的核心价值可以概括为三个“从”：从一次问答走向持续理解，从内容生成走向可信、可执行的学习资源，从统一推荐走向由真实学习证据驱动的动态路径。

我们希望问阶不仅能回答学生“这道题怎么做”，更能持续回答“我为什么不会、接下来学什么、怎样证明我已经学会”。

以上就是我们的项目演示，谢谢各位评委老师。

## 演示准备建议

- 提前登录并准备一份完整的学习画像，避免现场从空数据开始。
- 提前上传课程 PDF，并准备一个能稳定命中页码引用的问题。
- 多智能体生成建议使用预生成结果，现场重点展示协作轨迹和产物，控制等待时间。
- 路径中至少准备 2 个阶段、4 个 Todo，并完成其中 1～2 项，方便展示动态进度。
- 准备一组答题记录和错题，确保“评估—复盘—路径更新”的闭环能够直观看见。
- 若现场网络异常，按“首页—画像—知识库回答—资源结果—路径—评估”的顺序使用已有数据演示。
