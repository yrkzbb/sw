# 内容安全与防幻觉机制

系统采用多层防护，而不是只依赖提示词：

1. 浏览器输入检查：对凭证盗取、破坏性恶意软件、现实暴力操作、自伤操作和涉及未成年人的性内容进行拦截。
2. 服务端重复检查：`/api/chat` 在转发模型请求前再次检查最后一条用户消息，并限制消息数量和单条文本长度。
3. Prompt 注入识别：识别要求泄露系统提示词或覆盖系统指令的内容，并保持系统规则优先。
4. Agent 输出审核：各资源 Agent 的结果在进入资源池前执行内容安全检查；严重风险结果直接丢弃。
5. 学术风险标记：检测外部 URL、DOI、arXiv 编号、“研究表明”等引用线索及绝对化结论，标记为需要来源复核。
6. 来源区分：高校平台目录召回结果和 AI 生成内容使用不同徽标，不将模型记忆当作已检索事实。
7. 审核 Agent：独立检查主题一致性、来源标记、资源互补性和学习路径。
8. 失败降级：单个资源 Agent 超时或失败时自动重试一次，仍失败则跳过，不向用户展示不完整伪结果。

## 已知边界

- 本地规则不能识别所有隐晦表达，不能替代专业内容审核服务。
- 平台级目录核验不等于对每一门课程、每一篇论文进行了全文真实性验证。
- arXiv 等预印本平台内容不一定经过同行评议。
- 模型生成的代码、公式、答案和引用仍需通过测试、教材或原始文献复核。

## 推荐部署增强

面向公开生产环境时，建议增加服务商内容审核 API、URL 可达性检查、DOI/Crossref 校验、恶意文件扫描、速率限制、审计日志脱敏和人工申诉机制。
